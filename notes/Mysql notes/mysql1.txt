 四 整型列的可选属性
1 tinyint(M) usigned zerofill

M: 宽度（在0填充，也就是zerofill的时候才有意义） 不影响存储范围
unsigned:无符号类型（非负） 影响存储范围的
zerofill:0填充，（默认无符号） 规定一个长度，不够的位数用0填充

2 default 列的默认值 
 列可以声明默认值，并且推荐声明默认值
 not null default 0

 alter table class add age3 tinyint not null default 0; 默认初始值为0

 1 Float(M,D) (浮点型)

 orcale 和 sqlserver都不允许指定浮点型的小数位

1 存储范围不同 char 定长字符串 char(M)  0<=M<=255
        M:代表宽度，表示可容纳的字符数
  varchar 变长字符创 varchar(M) 0<=M<=65535(此处单位为字节)
        M同上

char 和 varchar的区别
2 实际占的空间不同
char定长型：M个字符，如果小于M个字符，实际占M个字符。
carchar变长型：M个字符，存的小于M个字符，设为N，实占N个字符 在存储时，会利用1到2个字节
               来记录到底存了多少字节。

3 对末尾的处理不一样 
对于char类型，存储时如果不够M个字符，内部会用空格补齐
              取出来时会把尾部的空格去掉
    varchar则因为有专门的记录所以不用
    所以如果用char类型存数据时，如果尾部有空格，则取出时会丢失数据（也就是尾部的空格）

4 速度上 定长快一些

5 如何选择 char varchar
  (1) 空间利用效率
 （2）速度

 还有一种 text类型 文本串 约2w到6w个字符 这种类型无法索引
 如果不是特别大不用用，而且不能用默认值

(name)

注意：
在开发中，很少用日期时间类型来表示一个需要精确到秒的列。
原因是，虽然日期时间类型能精确到秒，而且方便查看。
用时间戳来表示更方便计算

时间戳：是1970-01-01 00:00:00 到当前的秒数 用Int来存储
